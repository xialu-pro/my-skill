# Quick Start Writer Skill

快速入门文档是用户接触产品的第一入口，直接影响用户留存和转化。本skill帮助你写出高质量的快速入门文档。

## 工作流程

### 第一步：了解产品信息

询问用户以下关键信息：

| 维度 | 问题示例 |
|------|----------|
| 产品类型 | 是框架/工具/平台/语言/操作系统？ |
| 核心价值 | 用户用它能做什么？ |
| 路径维度 | 用户需要选择OS/架构/安装方式吗？有几层选择？ |
| 受众类型 | 开发者/运维/测试/算法工程师？受众是否多元？ |
| 安装必要性 | 能否先在线体验再安装？ |

### 第二步：选择范式

根据产品特点选择合适的范式：

```
Q1: 用户是否需要先理解概念才能操作？
├─ 是 → 开头加"概念概览"
│       ├─ 概念密集 → 【单页叙事型】
│       └─ 概念适中 → 【向导步骤型】+概念章节
└─ 否 → 继续 Q2

Q2: 存在多条路径吗？（OS/语言/环境/角色）
├─ 无 → 单路径
│       ├─ 工具链产品 → 【极简安装型】
│       └─ 工具型产品 → 【向导步骤型】
└─ 有 → 判断路径维度
        ├─ 1-2层 → 用选择器/检测 → 【分流枢纽型】或【极简安装型】
        ├─ 3-4层 → 判断路径总数
        │         ├─ ≤10条 → 尝试简化为【分流枢纽型】
        │         └─ ≥10条 → 【分层索引-向导型】
        └─ 主题独立性强 → 【索引目录型】

Q3: 安装是否可以推迟？
├─ 可推迟 → Sandbox/在线环境 → 【单页叙事型】
└─ 不可推迟 → 安装作为第一步
              ├─ 极简安装型：OS感知+单命令
              ├─ 向导步骤型：选择器+编号步骤
              └─ 分层索引-向导型：索引目录+各路径编号步骤
```

### 第三步：生成文档结构

根据选定的范式，生成对应的文档结构。

## 六种范式详解

### 1. 单页叙事型

**适用场景**：概念密集型产品，如React等前端框架

**核心特征**：
- 无安装墙，Sandbox直接运行
- 一条故事线贯穿，概念递进式讲解
- 每节：解释 + 可运行代码示例

**结构模板**：
```
首屏承诺盒："你将学到" + 预计用时 + 无需安装提示
概念梯子：基础 → 进阶 → 高级，每节配Sandbox
离场设计：教程路径 vs 概念学习路径
```

### 2. 向导步骤型

**适用场景**：工具型产品，如Docker

**核心特征**：
- 严格编号步骤
- 单一示例贯穿全部章节
- 每步有预期输出

**结构模板**：
```
首屏承诺："你将完成XXX"
Part 1: 概念概览
Part 2-N: 编号步骤（命令+预期输出）
Part N+1: 结构化离场
```

### 3. 分流枢纽型

**适用场景**：受众多元、无单一路径，如Kubernetes

**核心特征**：
- 首屏问题引导分流
- 每条路径独立完整
- 无单一推荐路径

**结构模板**：
```
首屏问题："你是学习环境还是生产环境？"
分流链接：各环境对应的快速路径
子章节链接：Minikube / kind / 云服务商等
```

### 4. 极简安装型

**适用场景**：工具链产品，如Rust

**核心特征**：
- 页面极短（5分钟通读）
- OS自动检测
- 单命令安装 + Hello World

**结构模板**：
```
OS检测 → 显示当前系统对应的安装命令
执行安装 → 列出安装内容
创建第一个项目 → 预期输出
三条学习路径：书 / 练习题 / 示例代码
```

### 5. 索引目录型

**适用场景**：文档体系庞大、主题独立性强，如Fedora

**核心特征**：
- 纯目录索引
- 无引导分流
- 用户自行判断选择

**结构模板**：
```
快速文档目录
安装与部署：下载/升级/卸载
核心功能：配置/网络/存储
常见场景：Web服务器/数据库/容器
```

### 6. 分层索引-向导型

**适用场景**：路径极多（≥3层决策），如NVIDIA CUDA

**核心特征**：
- 外层索引目录（无引导）
- 内层统一编号步骤
- 结构一致可复用

**结构模板**：
```
外层目录：OS → 架构 → 发行版 → 安装方式
内层步骤：Step 1-4（命令+预期输出）
每条路径独立但结构一致
```

## 开发者生态快速入门范式（推荐）

对于开发者生态文档（昇腾、鲲鹏、欧拉等），推荐使用收敛范式：

**开发者生态范式 = 分流引导 + 向导步骤 + 分层索引**

### 核心结构

```
┌─────────────────────────────────────┐
│ 首屏三层                              │
│  第一层：结果承诺                      │
│    "完成本指南后，你将能够..."          │
│  第二层：用户分流                      │
│    开发者/运维/测试                    │
│  第三层：环境选择器                    │
│    架构/OS选择                        │
├─────────────────────────────────────┤
│ 编号步骤                              │
│  Step 1-4：命令 + 预期输出            │
├─────────────────────────────────────┤
│ FAQ章节（覆盖高频问题）                │
├─────────────────────────────────────┤
│ 离场设计                              │
│  深入原理/完成任务/遇到问题            │
└─────────────────────────────────────┘
```

## 五条核心规律

所有快速入门文档必须遵守以下规律：

### 规律一：首屏必须回答"我能得到什么"

用户开始阅读前需要明确的结果承诺。

**模板**：
```markdown
> **完成本指南后，你将能够**：
> 1. ✅ [具体能力1]
> 2. ✅ [具体能力2]
> 3. ✅ [具体能力3]
>
> **预计用时**：约 XX 分钟
```

**反例**：以"本文档介绍了XXX的相关功能……"开头

### 规律二：个性化发生在用户第一个动作

复杂产品必须解决"我适用哪条路径"的问题。

**原则**：
- 选择器优于纯目录
- 自动检测优于手动选择
- 问题引导优于无引导

### 规律三：代码示例四个质量标准

| 标准 | 说明 |
|------|------|
| **可直接运行** | 复制即可执行，无需替换占位符 |
| **预期输出可见** | 每个命令旁边有预期输出 |
| **最小可用** | 不包含与当前步骤无关的参数 |
| **版本锁定** | 使用具体版本号，不用latest |

### 规律四：安装墙是设计决策

核心问题：需要先安装，还是可以先体验？

- 能通过Sandbox先体验 → 无安装墙设计
- 安装不可推迟 → OS感知 + 单条命令

### 规律五：结尾离场设计同等重要

回答"然后呢？"，不能使用泛化的"查看更多文档"。

**模板**：
```markdown
## 下一步

| 场景 | 推荐路径 |
|------|----------|
| **深入原理** | → [概念文档] |
| **完成任务** | → [进阶教程] |
| **遇到问题** | → [FAQ/社区支持] |
```

## 输出要求

根据用户需求和产品特点，输出以下内容：

1. **范式推荐**：推荐最适合的范式，说明理由
2. **结构大纲**：生成完整的章节结构
3. **完整模板**：生成可直接使用的Markdown模板，包含：
   - 首屏承诺
   - 用户分流（如有）
   - 环境选择器（如有）
   - 编号步骤（含占位符）
   - FAQ章节
   - 离场设计

4. **质量检查清单**：提醒用户检查5条规律是否满足

---

## 变体适配

根据路径维度，开发者生态范式有三种变体：

| 路径维度 | 变体 | 特点 |
|----------|------|------|
| 1层（单路径） | 简化版 | 首屏承诺 + 编号步骤 + FAQ + 离场 |
| 2层（架构/OS） | 标准版 | + 用户分流 + 选择器 |
| ≥3层 | 扩展版 | + 外层索引目录 + 内层编号步骤 |

---

## 质量检查清单

生成文档后，检查以下项：

- [ ] 首屏有明确结果承诺
- [ ] 预计用时已标注
- [ ] 多路径时有分流引导
- [ ] 步骤有编号（Step 1/2/3...）
- [ ] 每个命令有预期输出
- [ ] 代码示例可直接运行（无占位符）
- [ ] 版本号具体（不用latest）
- [ ] FAQ覆盖高频问题
- [ ] 离场设计有明确路径（不少于2条）
- [ ] 文档更新时间已标注
