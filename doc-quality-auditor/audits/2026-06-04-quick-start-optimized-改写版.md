# 快速入门

本文档以TaiShan 200服务器上安装 openEuler 为例，旨在指导用户快速地安装和使用openEuler操作系统，更详细的安装要求和安装方法请参考《[安装指南](../../../server/installation_upgrade/installation/installation_guide.md)》。

📝【AI 补写 · 需人工校验】**开始之前**：本文将带你完成「获取安装源 → 完整性校验 → 启动安装 → 图形化安装 → 查看系统信息」的完整流程，预计耗时约 30–60 分钟（不含镜像下载）。开始前请确认已具备以下前置条件：

* 一台满足[最小硬件要求](#最小硬件要求)的服务器（本文以 TaiShan 200 为例）；
* 可登录并操作服务器的 iBMC 管理界面；
* 可访问 openEuler 社区下载发布包与校验文件的网络环境。

## 获取安装源

请按以下步骤获取openEuler的发布包和校验文件：

1. 登录[openEuler社区](https://openeuler.org)网站。
2. 单击“下载”。
3. 单击“社区发行版”，显示版本列表。
4. 在版本列表的“openEuler 24.03 LTS SP3”版本处单击“前往下载”按钮，进入版本下载列表。
5. 根据实际待安装环境的架构和场景选择需要下载的 openEuler 的发布包和校验文件。
   1. 若为AArch64架构。
      1. 单击“AArch64”。
      2. 若选择本地安装，选择“Offline Standard ISO”或者“Offline Everything ISO”对应的“立即下载”将发布包 “openEuler-24.03-LTS-SP3-aarch64-dvd.iso”下载到本地。
      3. 若选择网络安装，选择“Network Install ISO”将发布包 “openEuler-24.03-LTS-SP3-netinst-aarch64-dvd.iso”下载到本地。
   2. 若为x86\_64架构。
      1. 单击“x86\_64”。
      2. 若选择本地安装，选择“Offline Standard ISO”或者“Offline Everything ISO”对应的“立即下载”将发布包 “openEuler-24.03-LTS-SP3-x86\_64-dvd.iso”下载到本地。
      3. 若选择网络安装，选择“Network Install ISO”将发布包 “openEuler-24.03-LTS-SP3-netinst-x86\_64-dvd.iso ”下载到本地。

> ![](./public_sys-resources/icon-note.gif) **说明：**
>
> * 网络安装方式的 ISO 发布包较小，在有网络的安装环境可以选择网络安装方式。
> * AArch64架构的发布包支持UEFI模式，x86\_64架构的发布包支持UEFI模式和Legacy模式。

📝【AI 补写 · 需人工校验】**术语说明**：UEFI（统一可扩展固件接口）与 Legacy（传统 BIOS 引导）是两种不同的固件引导模式，决定服务器如何加载并启动操作系统；选择与服务器固件设置匹配的发布包可避免无法引导的问题。

## 发布包完整性校验

> ![](./public_sys-resources/icon-note.gif) **说明：**
> 本章节以AArch64架构的发布包完整性校验为例，x86\_64架构的发布包完整性校验的操作方法相同。

### 简介

为了确认软件包在传输过程中由于网络原因或者存储设备原因是否出现下载不完整的问题，在获取到软件包后，需要对软件包的完整性进行校验，通过了校验的软件包才能部署。

这里通过对比校验文件中记录的校验值和手动方式计算的iso文件校验值，判断软件包是否完整。若两个值相同，说明iso文件完整，否则，iso完整性被破坏，请重新获取iso发布包。

### 前提条件

在校验发布包完整性之前，需要准备如下文件：

* iso文件：openEuler-24.03-LTS-SP3-aarch64-dvd.iso

* 校验文件：ISO对应完整性校验值，复制保存对应的ISO值

### 操作指导

文件完整性校验操作步骤如下：

1. 计算文件的sha256校验值。执行命令如下：

   ```sh
   # sha256sum openEuler-24.03-LTS-SP3-aarch64-dvd.iso
   ```

   ⚠️【待人工确认 · 问题编号：N-007】上方代码块为原文内容，命令前的 `#` 与语言标识 `sh` 按代码块约束保留未改动。建议人工补充一行预期输出示例（形如 `<64位十六进制校验值>  openEuler-24.03-LTS-SP3-aarch64-dvd.iso`），以便用户对照确认。

   命令执行完成后，输出校验值。

2. 对比步骤1计算的校验值与对刚刚复制的SHA256的值是否一致。

   如果校验值一致说明iso文件完整性没有被破坏，如果校验值不一致则可以确认文件完整性已被破坏，需要重新获取。

进入图形化安装界面后，按如下步骤进行安装。

1. 设置安装语言，默认为英语，用户可根据实际情况进行调整，如[图6](#fig874344811484)所示，选择“中文”。

2. 在安装概览界面，根据实际情况设置各配置项。

   > ![](./public_sys-resources/icon-note.gif) **说明：**

   > * 配置项有告警符号的，表示用户必须完成该选项配置后，告警符号消失，才能进行下一步操作。
   > * 配置项无告警符号的，表示该配置项已有默认配置。
   > * 所有配置项均无告警符号时用户才能单击“开始安装”进行系统安装。

   **图 7**  安装概览\
   ![图 7 安装概览](./figures/Installation_Overview.png)

   1. 选择“软件选择”，设置“软件选择”配置项。

      用户需要根据实际的业务需求，在左侧选择一个“最小安装”，在右侧选择安装环境的附加选项，如[图8](#fig1133717611109)所示。

      **图 8**  软件选择\
      ![图 8 软件选择](./figures/choosesoftware.png)

      > ![](./public_sys-resources/icon-note.gif) **说明：**
      >
      > * 在最小安装的环境下，并非安装源中所有的包都会安装。如果用户需要使用的包未安装，可将安装源挂载到本地制作repo源，通过DNF工具单独安装。
      > * 选择“虚拟化主机”时会默认安装虚拟化组件qemu、libvirt、edk2，且可在附加选项处选择是否安装ovs等组件。

      设置完成后，请单击左上角“完成”返回“安装概览”页面。

   2. 选择“安装目的地”，设置“安装目的地”配置项。

      在安装位置页面中，您可以选择计算机中的本地可用存储设备。

      > ![](./public_sys-resources/icon-notice.gif) **须知：**
      >
      > * 由于很多服务器BIOS内置NVMe驱动程序版本较低，不支持NVMe的数据保护特性（数据保护：将磁盘扇区格式化为512+N或4096+N字节）。所以，在选择合适的存储介质时，建议不要选择开启数据保护特性的NVMe SSD存储介质作为系统盘，否则可能出现操作系统无法引导等问题。
      > * 用户可以选择优先咨询服务器厂商关于BIOS是否支持开启数据保护特性的NVMe磁盘作为系统盘。如果您无法确认BIOS是否支持，则不推荐使用NVMe安装操作系统，或者选择关闭NVMe盘的数据保护功能实现操作系统安装。

      您还需要进行存储配置以便对系统分区。您可以手动配置分区，也可以选择让安装程序自动分区。如果是在未使用过的存储设备中执行全新安装，或者不需要保留该存储设备中任何数据，建议选择“自动”进行自动分区。如[图9](#fig153381468101)所示。

      **图 9**  安装目标位置\
      ![图 9 安装目标位置](./figures/Target_installation_position.png)

      > ![](./public_sys-resources/icon-note.gif) **说明：**
      >
      > * 在进行分区时，出于系统性能和安全的考虑，建议您划分如下单独分区：/boot、/var、/var/log 、/var/log/audit、/home、/tmp。
      > * 系统如果配置了swap分区，当系统的物理内存不够用时，会使用swap分区。虽然 swap分区可以增大物理内存大小的限制，但是如果由于内存不足使用到swap分区，会增加系统的响应时间，性能变差。因此在物理内存充足或者性能敏感的系统中，不建议配置swap分区。
      > * 如果需要拆分逻辑卷组则需要选择“自定义”进行手动分区，并在“手动分区”界面单击“卷组”区域中的“修改”按钮重新配置卷组。

      **密码复杂度**&#x20;

      用户设置的root用户密码或新创建用户的密码均需要满足密码复杂度要求，否则会导致密码设置或用户创建失败。设置密码的复杂度的要求如下：

      2. 口令至少包含大写字母、小写字母、数字和特殊字符中的任意3种。
      3. 口令不能和账号一样。
      4. 口令不能使用字典词汇。

         > ![](./public_sys-resources/icon-note.gif) **说明：**
         > 在已装好的openEuler环境中，可以通过`cracklib-unpacker /usr/share/cracklib/pw_dict > dictionary.txt`命令导出字典库文件dictionary.txt，用户可以查询密码是否在该字典中。

## 查看系统信息

系统安装完成并重启后直接进入系统命令行登录界面，输入安装过程中设置的用户和密码，进入openEuler操作系统，查看如下系统信息。若需要进行系统管理和配置操作，请参考《[管理员指南](https://openeuler.org/zh/docs/24.03_LTS_SP3/server/administration/administrator/viewing_system_information.html)》。

* 查看系统信息，命令如下：

  ```sh
  cat /etc/os-release
  ```

  📝【AI 补写 · 需人工校验】预期输出：回显包含 `NAME="openEuler"`、`VERSION="24.03 (LTS-SP3)"` 等字段，可据此确认系统版本。

* 查看系统相关的资源信息。

  查看CPU信息，命令如下：

  ```sh
  # lscpu
  ```

  查看内存信息，命令如下：

  ```sh
  # free
  ```

  查看磁盘信息，命令如下：

  ```sh
  # fdisk -l
  ```

* 查看IP地址，命令如下：

  ```sh
  # ip addr
  ```

## 常见问题（FAQ）

**Q1：本地安装和网络安装该如何选择？**

📝【AI 补写 · 需人工校验】网络安装方式的 ISO 发布包较小，适合在有网络的安装环境使用；若安装环境无网络，请选择 Offline Standard ISO 或 Offline Everything ISO 进行本地安装。（依据「获取安装源」章节说明归纳）

**Q2：完整性校验时两个校验值不一致怎么办？**

📝【AI 补写 · 需人工校验】校验值不一致说明 ISO 文件完整性已被破坏，需重新获取 ISO 发布包后再次校验。（依据「发布包完整性校验」章节说明归纳）

**Q3：可以不划分单独分区直接安装吗？**

📝【AI 补写 · 需人工校验】可以选择「自动」分区完成安装；但出于系统性能与安全考虑，建议单独划分 /boot、/var、/var/log、/var/log/audit、/home、/tmp 等分区。（依据「安装目的地」章节说明归纳）

## 故障排查

📝【AI 补写 · 需人工校验】本节依据原文已有的「说明/须知」内容归纳常见问题与应对，具体处理以产品实际行为为准。

| 现象 | 可能原因 | 处理建议 |
|---|---|---|
| ISO 完整性校验值不一致 | 下载过程中网络或存储原因导致文件损坏 | 重新获取 ISO 发布包后再次执行 sha256 校验（见「发布包完整性校验」） |
| 安装后操作系统无法引导 | 使用了开启数据保护特性的 NVMe SSD 作为系统盘，而 BIOS 内置 NVMe 驱动版本较低 | 咨询服务器厂商 BIOS 是否支持；不确定时不推荐用 NVMe 安装，或关闭 NVMe 盘数据保护功能后再安装（见「安装目的地」须知） |
| 引导界面无法用键盘选择启动项、按 Enter 无响应 | 物理机键鼠未复位 | 单击 BMC 界面上的鼠标控制图标，设置「键鼠复位」（见「启动安装」步骤 8 说明） |
| 密码设置或用户创建失败 | 密码不满足复杂度要求 | 按密码复杂度要求重设：长度≥8、含大小写/数字/特殊字符任意 3 种、不与账号相同、不使用字典词汇（见「根密码」章节） |

💡【写作建议 · 待补充】可进一步补充安装介质无法挂载、网络安装源不可达等高频问题的官方处理建议。
